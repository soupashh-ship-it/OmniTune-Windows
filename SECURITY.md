# Security Policy

## Reporting a Vulnerability

Do not publish exploit details, tokens, credentials, or private user data in a public issue.

If GitHub private vulnerability reporting is available on this repository, use it. If it is not available, open a minimal public issue that says you need a private security reporting channel, without including sensitive details.

## Sensitive Data

Reports should never include:

- access tokens
- cookies
- API keys
- private keys
- signing certificates
- local keystores
- personal account credentials
- downloaded private media

## Scope

Relevant issues include vulnerabilities in:

- desktop persistence
- download handling
- local file handling
- provider authentication/cookie handling
- release artifacts
- installer behavior
- dependency supply chain
